Miku IM Server - Development Notes

2025-05-24 | Phase 0 DONE

Architecture Design Complete

Completed full analysis of OpenIM Server (Go, 12 microservices, 100+ API endpoints, 47K LOC). Designed C rewrite architecture with: Arena+Slab memory pools, work-stealing thread pool, ucontext stackful coroutines, epoll/kqueue/IOCP cross-platform I/O, custom binary RPC with PB codec.

Foundation + Runtime + I/O Layer Complete

  • Root CMakeLists.txt with C23 standard, platform detection, optional dependencies
  • 9 sub-CMakeLists.txt for all modules
  • 12 service entry points in cmd/
  • Convenience Makefile (make debug/test/clean)
  • Foundation layer: 15 complete modules
  • Runtime layer: coroutine, threadpool, scheduler, channel, timer
  • I/O abstraction: epoll (edge-triggered) with socketpair test
  • Test framework header (miku_test.h)
  • 20 tests, 67 assertions, all passing
2025-05-24 | Phase 1F DONE

Protocol Layer: HTTP + JSON

  • miku_http.h/c - HTTP/1.1 request parser (method, path, query string, headers, body)
  • miku_http_server.h/c - HTTP server with route matching, non-blocking accept/read via miku_io
  • miku_json.h/c - JSON parser (object, array, string, int, double, bool, null) + builder + stringify
  • HTTP tests: parse GET with query, parse POST with body, incomplete detection, method enum, response serialize, status text, server ping-pong integration test
  • JSON tests: parse object, parse array with mixed types, primitives (true/false/null/negative), nested objects+arrays, stringify roundtrip, builder API, missing key handling
  • 27 tests total (11 foundation + 9 runtime + 8 HTTP + 8 JSON), all passing
  • Old stub files (miku_protocol.h/c) removed
